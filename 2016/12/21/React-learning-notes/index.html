React Learning Notes

Posted on 2016-12-21 |

ReactJS

本系列文主要用于记录一名front-end newbie 学习React的路程，主要包括一些基础要点的总结，和知识细节的分析。如果觉得过于面面俱到，那说明这篇文章已经配不上你的技术水平了，请绕路。

1)

React is the trend.

什么是React的呢？学习React之前，要先试着理解它的定义。引用官网的介绍：”A JAVASCRIPT LIBRARY FOR BUILDING USER INTERFACES”。

首先，React是一个js库(Library)，区别于框架(Framework)。框架但更重量级一些，通过创造一些需要你去遵守的规则和范例，为你提供一整套的解决方案。例如AngularJS,它就是一个框架，你在使用它的时候，需要按照它设计好的方式来写代码，因此你是受控于这个框架内的。 而库呢，可以简单的认为就是一些方法的集合(a collection of functionality)。 例如jQuery，当你需要它提供的一些方法时，你直接调用即可。此时就是你在控制这个库。
什么意思呢？

  • 框架和库的区别最重要在于控制权是否掌握在开发者手中(Inversion of Control)。
  • 也就是说，库是由你去调用的，因此你控制它。但框架呢，控制权是反过来的，框架调用你。 (This is called the Hollywood Principle: Don’t call Us, We’ll call You.)
    Reference from Stackoverflow

其次，React是一个针对开发UI的库。它不具备完整的MVC架构，而只是其中的V层部分。它关心：

  • 监视View层DOM的变化，显示并且更新DOM
  • 相应事件

它不关心：

  • 处理Ajax
  • 路由
  • 数据存储

其设计理念十分简洁，高效。之所以高效，是因为它创建的是虚拟DOM(virtual DOM),而非直接操作DOM，运用高效的DOM Diff算法，最小化页面重绘。
